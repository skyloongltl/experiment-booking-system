1. 角色
    * 学生
    * 教师
    * 管理员
    
2. 信息结构
    * 用户　－－　模型名称　Student
    
3. 动作
    * 创建　Create
    * 查看　Read
    * 编辑　Update
    * 删除　Delete
    
4. 用例
    1. 学生
        * 学生可以查询本学期可选的实验
        * 学生可以预约本学期可以预约的实验
        * 学生可以修改密码
        
    2. 教师
        * 教师可以查询本学期可选的实验
        * 教师可以预约教那个实验
        * 教师可以修改密码
        
    3. 管理员
        * 管理员可以管理（即增删查改）学校信息
        * 管理员可以管理学期信息
        * 管理员可以管理实验室信息
        * 管理员可以管理学生和教师的信息
        * 管理员可以管理实验的信息
